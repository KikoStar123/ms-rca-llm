# 流程图说明文本（配图注释）

更新时间：2026-01-23

## 1. 总体流程图说明
系统从故障事件触发开始，依次完成多源观测数据接入、时间窗对齐与证据聚合；小模型生成 Top‑K 根因候选，大模型在候选基础上输出根因与故障类型，并给出可解释说明；最终结果经统一 Schema 格式化并展示。

## 2. 数据准备流程图说明
以故障清单为入口，在固定时间窗内聚合日志、指标与链路数据，生成证据摘要；证据与标签组合形成 SFT 样本，用于大模型监督微调。

## 3. 训练流程图说明
SFT 阶段先让模型掌握输出格式与基本定位能力；GRPO 阶段引入奖励约束，强调输出稳定性、证据一致性与格式合规性。

## 4. 推理流程图说明
运行时由小模型先给出 Top‑K 候选，大模型在候选中进行解释性推理与分类，输出可直接展示的结构化结果。
