# 系统工程实现细化（接口/顺序/产出）

更新时间：2026-01-23

## 1. 模块顺序（从输入到输出）
1) `data_ingest`：原始日志/指标/链路入库  
2) `window_align`：时间窗对齐与过滤  
3) `evidence_agg`：证据聚合（min/max/avg/成功率）  
4) `candidate_ranker`：小模型 Top‑K 候选  
5) `llm_reasoner`：大模型解释与分类  
6) `result_formatter`：结构化输出与展示

## 2. 输入输出接口（建议）

### 2.1 data_ingest
**输入：**
- 原始 CSV / 日志文件
**输出：**
- 标准化记录（含 `timestamp`, `cmdb_id`, `type`, `value`）

### 2.2 window_align
**输入：**
- 故障事件（log_time、window_before/after）
- 标准化记录
**输出：**
- 事件窗口内记录集合

### 2.3 evidence_agg
**输入：**
- 窗口内记录集合
**输出：**
- evidence 摘要（min/max/avg/成功率）

### 2.4 candidate_ranker
**输入：**
- evidence 摘要
**输出：**
- Top‑K 候选列表（cmdb_id）

### 2.5 llm_reasoner
**输入：**
- fault_event + evidence + top_candidates
**输出：**
- root_cause_component
- fault_type
- explanation（可选）

### 2.6 result_formatter
**输入：**
- 结构化结果
**输出：**
- JSON/表格/前端展示结构

## 3. 最小化数据流示例
```
fault_event -> window_align -> evidence_agg -> candidate_ranker -> llm_reasoner -> result_formatter
```

## 4. 现有对应脚本
- evidence 构建：`pipeline/build_sft_samples.py`
- Top‑K 候选：`pipeline/small_model_rootcause_weighted_topk.py`
- LLM 输入：`pipeline/build_llm_inputs.py`
