# 章节要点与贡献点（可直接引用）

更新时间：2026-01-23

## 1. 引言
**要点：**  
微服务架构中故障传播链路复杂、依赖关系强、告警噪声高，导致传统规则/经验驱动的排障方案难以在多源观测数据中快速定位根因并给出稳定结论。  
**关键词：** microservices, RCA, observability, log/metric/trace, diagnosis cost  

## 2. 相关工作
**要点：**  
现有研究在工具增强推理、多智能体协作、以及小模型与大模型协同方面取得进展，但在稳定性与可迁移性方面仍存在不足。  
**关键词：** tool-augmented reasoning, multi-agent RCA, small-large model collaboration  

## 3. 数据集与数据准备
**要点：**  
基于 AIOps2020 数据，构建统一时间窗对齐与证据聚合流程，形成可直接用于 SFT 的结构化样本。  
**关键词：** window alignment, evidence aggregation, SFT dataset  

## 4. 方法设计
**要点：**  
提出“大小模型协同”框架：小模型完成候选根因 Top‑K 排序，大模型完成根因判别与故障类型分类，并输出可解释结果。  
**关键词：** Top‑K candidate, LLM reasoning, fault type classification  

## 5. 系统工程实现
**要点：**  
实现端到端流程：数据接入→证据聚合→候选生成→大模型推理→结果展示，并制定统一输出 Schema。  
**关键词：** pipeline orchestration, schema, interface  

## 6. 实验与评估
**要点：**  
小模型基线表明加权策略能稳定提升 Top‑K 命中率；LLM 输入样本分布均衡，输出格式合规性良好。  
**关键词：** hit@k, weighted baseline, schema validation  

## 7. 贡献点总结（可直接列在摘要/结论）
1) 构建多源观测数据对齐与聚合流程，形成可直接用于大模型微调的 RCA 数据集。  
2) 提出大小模型协同定位框架，利用小模型生成候选根因，大模型完成解释性判别与分类。  
3) 设计并实现端到端 RCA 流程与统一输出 Schema，验证系统的可复现性与可扩展性。
