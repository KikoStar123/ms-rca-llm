## 仓库内已落地的支撑项（2026-04）

- **路径**：`config.py` 支持任意克隆目录；**推荐**设置 `GRADUATION_PROJECT_ROOT` 指向仓库根（本机/服务器均可）；模型路径 `QWEN_MODEL_PATH`。PowerShell：`. .\scripts\local_env.ps1`
- **划分产物**：`output/train_split.jsonl`、`output/eval_split.jsonl`、`output/split_manifest.json`（已与训练脚本 shuffle 逻辑一致）。
- **可复现参数**：`config/experiment_defaults.yaml`。
- **数据划分**：`pipeline/split_dataset.py`（与 `train_qwen2_5_7b_qlora_demo` 的 shuffle+切分一致）。
- **JSON 解析与失败原因**：`pipeline/json_parse_utils.py`；训练脚本已改为复用该模块。
- **离线指标**：`pipeline/evaluate_predictions_jsonl.py`；存在 `output/eval_split.jsonl` 时会自动作为金标准合并（亦兼容手动 `--gold`）。
- **训练划分**：`train_qwen2_5_7b_qlora_demo.py` 默认读取 `output/train_split.jsonl` / `eval_split.jsonl`；`USE_FIXED_SPLIT=0` 恢复旧 shuffle 切分。8-bit 量化时已按 `docs/CRITICAL_FIX.md` 禁用 `gradient_checkpointing`。
- **演示推理**：`pipeline/inference_cli.py`（`--mock` 无需 GPU）；样例 `research/demo_inference_sft_sample.json`。
- **流程说明**：`python pipeline/run_rca_pipeline.py list`。
- **GRPO**：`pipeline/large_model/grpo_rewards.py` + `grpo_train_skeleton.py`（需在 GPU 环境接入 TRL）。

仍需你本地/服务器完成的：**在提升指标的前提下跑通 SFT、再接入 GRPO、补论文实验表**。

## 一、优先级说明

1. **先做**：能把指标和可解析率明显拉起来、论文里能画表对比的部分（数据、SFT、小模型召回、评估闭环）。  
2. **后做**：在 SFT 基线可靠后再上 **GRPO**（避免在弱基线上堆 RL）。  
3. **并行**：系统工程（接口、配置、一键脚本）可与实验迭代穿插，避免最后突击。

## 二、数据与样本构建

| 序号 | 工作项 | 说明 |
|------|--------|------|
| D1 | 统一故障事件与时间窗口 | 与开题一致：以故障时间为中心，明确 `window_start/window_end` 规则（如 ±15min），全样本口径一致。 |
| D2 | 多源证据对齐与清洗 | 平台指标、调用链、业务、容器等在同一窗口内对齐；处理缺失、延迟、采样频率不一致。 |
| D3 | 证据摘要字段固化 | 按组件聚合的统计量（均值/极值/失败率/慢调用率等）与 `pipeline` 中模板一致，避免训练/推理分布漂移。 |
| D4 | 数据质量与分布 | 抽样检查标签噪声、类别不均衡、字段缺失；必要时重采样或加权，并记录数据版本号。 |
| D5 | 训练/验证/测试划分 | 固定随机种子与划分方式，保证可复现；所有对比实验共用同一划分。 |

## 三、小模型（Top-K 候选召回）

| 序号 | 工作项 | 说明 |
|------|--------|------|
| S1 | 实现或补全 `pipeline/small_model` | 可解释加权打分 `score(s)=Σ w_i·z_i(s)`，输出 Top-K 候选列表。 |
| S2 | 候选覆盖指标 | 统计 **Hit@K**（真实根因是否落入 Top-K），作为小模型与 K 的调参依据。 |
| S3 | K 与消融 | 对比不同 K 对端到端成本、大模型命中与解析率的影响；论文中可写消融。 |
| S4 | 可选：触发策略 | 告警风暴场景下的归并/降噪、置信度阈值（开题 4.3 已提及），有时间再做。 |

## 四、大模型 SFT（QLoRA）

| 序号 | 工作项 | 说明 |
|------|--------|------|
| L1 | 提示词与输出约束 | 明确「仅在候选列表中选根因」、单行 JSON、四字段 schema；与训练数据一致。 |
| L2 | 训练超参网格 | 对学习率、epoch、batch/累积步数、LoRA 配置等做小规模对比，固定**一套**可复现配置写入 `config` 或 README。 |
| L3 | 训练稳定性 | 关注 4-bit 量化下 loss、梯度；保留完整日志（`logs/`），便于论文描述异常与排查。 |
| L4 | 基线对比 | 报告「微调前 vs 微调后」在同一评估集上的 **Top-1/3/5、各字段 Acc、Full Match、Parse Rate**；当前 `v2_doc` 显示轻量 SFT 提升有限，需通过数据与模板迭代拉高差距。 |
