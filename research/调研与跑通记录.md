# 调研与跑通记录（AIOps2020）

更新时间：2026-01-23

## 1. 文献调研要点（Paper1-5）

### Paper1：AgentThink（VLM + Tool-augmented CoT + SFT/GRPO）
- 关键点：两阶段训练（SFT + GRPO）提升推理稳定性与工具调用一致性。
- 贡献：结构化推理链、工具调用奖励设计、推理一致性评估指标。
- 与课题关联：为“大模型协同 + GRPO优化”提供训练范式与奖励设计思路。

### Paper2：MABC（多智能体 + 区块链式投票）
- 关键点：多智能体分工、投票去幻觉、依赖链路与传播路径追踪。
- 贡献：Agent Workflow 分工流程，投票机制提升可靠性。
- 与课题关联：可用于系统工程中的“多智能体分析编排”和“结果一致性校验”模块设计。

### Paper3：LasRCA（LLM + 小模型协作，一样本RCA）
- 关键点：小模型先定位/分类，LLM补充解释与高混淆样本标注。
- 贡献：小模型+LLM闭环，降低成本，缓解标注稀缺。
- 与课题关联：正对应“大小模型协同”，可作为论文主线核心思路。

### Paper4：FlowXpert（知识库 + 多智能体协同 + PPO/DPO）
- 关键点：知识库构建（向量+图），Planner/Scorer双智能体协同优化。
- 贡献：流程编排优化、评估指标（STEPScore）、偏好数据生成。
- 与课题关联：可借鉴为系统工程中的“流程编排器 + 评估器”设计。

### Paper5：RCAgent（工具增强LLM自主RCA）
- 关键点：工具化数据采集、观测压缩、稳定性控制、轨迹一致性。
- 贡献：自主RCA框架与工业落地案例。
- 与课题关联：支撑“系统工程实现”的工具调用与稳定性机制设计。

## 2. 与课题三部分的映射

### 2.1 数据准备与SFT
- 借鉴：LasRCA 的“LLM辅助小模型标注”，AgentThink 的“结构化推理数据”。
- 可行方案：从 AIOps2020 中抽取【告警窗口】→对齐日志/指标/链路 → 生成“证据+结论”样本，用于SFT。

### 2.2 GRPO优化
- 借鉴：AgentThink 的 GRPO 奖励结构（最终答案、步骤一致性、工具使用质量）。
- 可行方案：在 SFT 基础上加入 GRPO，奖励稳定、可解释、可复核的推理链与根因判断。

### 2.3 系统工程实现
- 借鉴：MABC/FlowXpert/RCAgent。
- 可行方案：构建“数据接入→特征聚合→候选根因（小模型）→大模型解释与分类→结果展示”的流水线。

## 4. 可直接执行的下一步

1) **数据对齐**：以故障清单中的 `log_time` 为中心，聚合固定窗口内的日志/指标/调用链数据。  
2) **候选根因（小模型）**：先做轻量分类/排序，输出 Top-k 根因组件。  
3) **解释与分类（大模型）**：基于Top-k与证据，生成“根因+故障类型+证据链”。  
4) **SFT样本生成**：保存为 `{input: 观测证据, output: 根因+类型+解释}`。  
5) **GRPO阶段**：在SFT基础上加奖励约束（稳定性、证据引用、格式一致）。
