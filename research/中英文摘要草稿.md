## 中文摘要
面向微服务架构中故障传播链路复杂、组件依赖强与告警噪声高的问题，本文设计并实现一种基于大小模型协同的故障根因定位与故障类型分类方法。首先，构建多源观测数据（日志、指标、链路追踪）的时间窗对齐与证据聚合流程，形成可用于大模型监督微调的结构化样本。其次，采用小模型生成 Top‑K 候选根因，大模型在候选基础上完成解释性判别与类型分类，并为输出定义统一 Schema。实验基线结果表明，加权候选策略在 Top‑K 命中率上优于纯频次方案，结构化输出具有良好合规性。最后，构建端到端流程与最小接口示例，为系统落地提供可复现实现路径。

## English Abstract
To address complex fault propagation, strong component dependencies, and noisy alerts in microservice systems, this work designs and implements a small‑large model collaboration framework for root cause localization and fault type classification. We first build a time‑window alignment and evidence aggregation pipeline over multi‑source observability data (logs, metrics, traces), producing structured samples for supervised fine‑tuning. A small model generates Top‑K candidate root causes, while a large model performs explainable reasoning and fault type classification with a unified output schema. Baseline results show that weighted candidate ranking improves Top‑K hit rates over frequency‑only heuristics, and the structured outputs pass schema validation. An end‑to‑end pipeline and minimal interface examples are provided for reproducible deployment.
